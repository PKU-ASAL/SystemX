# SysArmor v0.1.0

**发布日期**: 2025年9月15日  
**版本类型**: 首个正式版本

## 🎉 新功能

### 🔧 SysArmor Manager 后端
完整的后端 API 服务，提供全面的管理功能：
- **设备管理**: Collector 注册、心跳监控、元数据管理
- **服务集成**: 统一管理 Kafka、Flink、OpenSearch 的 API 接口
- **健康监控**: 20+ 项系统健康检查和服务状态监控
- **资源管理**: 动态生成脚本和配置文件
- **RESTful API**: 完整的 Swagger 文档，访问 `/swagger/index.html`

### 🖥️ SysArmor Manager 前端 UI
现代化的 Web 管理界面：
- **实时仪表板**: 系统概览和实时指标展示
- **设备管理**: 基于 Web 的 Collector 管理界面
- **事件监控**: 实时事件流可视化
- **系统健康**: 交互式健康状态监控
- **响应式设计**: 支持移动端访问

### 🤖 SysArmor ML 服务
机器学习威胁检测基础设施：
- **训练管道**: NODLINK 模型训练和验证
- **实时推理 API**: 高性能威胁检测推理服务
- **多模型支持**: 按 Collector 的模型管理
- **威胁检测**: 自动化异常检测和告警生成
- **模型管理**: 负载均衡和模型缓存

## 🔧 技术架构

### 系统架构
- **微服务设计**: Manager + Middleware + Processor + Indexer 四大模块
- **数据流**: auditd → Vector → Kafka → Flink → OpenSearch
- **实时处理**: 基于 NODLINK 算法的实时威胁检测
- **部署方式**: 支持单机和分布式部署

### 性能指标
- **数据摄入**: ~1,000 事件/秒
- **处理延迟**: <1 秒端到端处理
- **事件过滤**: 支持 22 种 NODLINK 标准事件类型
- **资源需求**: 推荐 4 核 CPU，16GB 内存

### 技术栈
- **后端**: Go 1.24, Python 3.11
- **前端**: React 18, TypeScript
- **流处理**: Apache Flink 1.18
- **消息队列**: Apache Kafka 3.4
- **搜索引擎**: OpenSearch 2.11
- **数据库**: PostgreSQL 15

## 🚀 快速开始

```bash
# 快速部署
git clone https://github.com/sysarmor/sysarmor.git
cd sysarmor
make init && make deploy

# 验证安装
make health
curl http://localhost:8080/health

# 运行测试
./tests/test-system-health.sh
./tests/test-kafka-producer.sh sysarmor-agentless-samples.jsonl
```

## 🔍 已知问题

- Web UI 需要现代浏览器支持 (Chrome 90+, Firefox 88+)
- 大数据量处理 (>10K 事件/秒) 可能需要额外调优
- 分布式部署需要手动配置防火墙规则
